# Cat Activity Tracker (XIAO nRF52840 sence + LIS2DH12)

Проект трекера активности кота на базе XIAO и акселерометра LIS2DH12.
Прошивка считает активность, шаги (оценочно), метры, прыжки, хранит статистику по часам и отдает данные по BLE.

Проект вдохновлен нигилизмом и преследует несколько целей:
- Минимализм
- Высокая автономность
- Безопасность кота
- Избегает чрезмерного количества ненужных фичей
- Дает удовольствие от процесса
- Дает возможность изучать здоровье кота

## Что делает система

- Работает в двух режимах датчика:
  - `WAKE`: низкое потребление, ожидание движения
  - `ACTIVE`: чтение ускорений и анализ движения
- Выделяет сегменты движения и фильтрует шум.
- Оценивает:
  - `steps` (через длину шага)
  - `meters` (через длительность сегмента и оценку скорости)
  - `activePoints`
  - `jumps`
- Сохраняет состояние во Flash (`LittleFS`) и ведет почасовые бакеты.
- Отдает статистику по BLE-команде `GET_STATS`.

## Логика работы (процесс)

1. **Старт устройства**
- Инициализация LED, батарейного АЦП, файловой системы, акселерометра, BLE.
- Загрузка сохраненного состояния или запуск с дефолтными параметрами.

2. **Режим WAKE**
- LIS2DH12 работает с пониженной частотой и генерирует `INT1` при активности.
- При срабатывании IRQ устройство переходит в `ACTIVE`.

3. **Режим ACTIVE**
- Чтение сэмплов ускорения.
- Расчет динамики (`dyn`) относительно сглаженной гравитации.
- Подтверждение старта сегмента движения (`pending start` -> `movement ongoing`).
- Учет прыжков по `zDyn` с hysteresis и cooldown.
- Завершение сегмента по тишине (`MOTION_END_GAP_MS`) или по `MAX_MOVEMENT_SEGMENT_MS`.

4. **Подсчет метрик сегмента**
- Проверка фильтров качества сегмента (минимальная длительность/насыщенность движением).
- Оценка скорости из среднего `avgDyn`.
- Расчет метров и шагов.
- Запись в текущий часовой бакет и в общие счетчики.

5. **Фоновые задачи**
- Периодическое измерение батареи через делитель + MOSFET-ключ.
- Периодическое сохранение состояния в Flash.
- BLE-обработка команд (`TIME_SYNC`, `GET_STATS`, `RESET_STATS`, `DISCONNECT`, `DEBUG_CHANGE`).

## Примечания

- Текущие коэффициенты в прошивке откалиброваны под кота (не под человека).
- Самая честная метрика - активность
- Для корректной почасовой статистики после подключения по BLE рекомендуется прислать `TIME_SYNC:<unix_ts>`.
